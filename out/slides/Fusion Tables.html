    about our data

  • data came from 2 differen sources with different schema
  • data was in esri format and is points and lines
  • fusion tables is a table based schema where all rows must have the same fields
  • geometry is in a kml format
  • esri considers points and line features to be totally seperate never mixable
  • kml pretty much considers lines to be long points

    • importing into fusion tables

  • if all you have is points, it is pretty easy to get your data in
  • just make sure your data is in wgs84 and save it as a csv with a lat and a lng collumn
  • if it isn't, then it's trickier

    • method 1. from esri

  • you can import into fusion tables kml files upto 100 MiB
  • but it takes the atributes from the kml extended data
  • esri refuses to aknowledge that kml files have an extrended data atribute
  • make sure your data has a primary key of some sort
  • make sure that is the display field
  • then export it as a kml, and also export the atribute table as a csv
  • upload them both the fusion tables as seperate tables
  • do a merge (aka a join)

    • method 2. qgis

  • or you can save it as a shape file
  • open it with quantum gis
  • save it as a kml
  • you can just import that into fusion tables
  • this is where my g/fs old laptop came in, i had whiped it and installed ubuntu and quantum gis on it.
